# World-First Positioning Note

Date: 2026-04-04

## Core Direction

Worldbuilding Desk should lean harder into being **world-first**, not merely plot-first with lore attached.

For the kinds of projects this app is strongest at, especially:

- LitRPG
- RPG-adjacent fiction
- progression fantasy
- world-heavy speculative fiction

the product should assume that these carry weight comparable to plot:

- world structure
- character depth
- canon continuity
- systems/mechanics when relevant

## Working Position

The app is not just:

- a drafting tool
- a plotting tool
- an AI writing surface with some reference panels

It should feel like:

- a writing desk for stories where the world matters as much as the plot

## Why This Matters

There appears to be whitespace between:

- tools that emphasize drafting/AI generation
- tools that emphasize outlining
- tools that treat lore/worldbuilding as secondary support material

The opportunity here is:

- worldbuilding as a first-class creative surface
- character and canon as co-equal to plot
- systems and state as first-class where the genre needs them

## Product Implications

This suggests several strategic filters:

### 1. Do not subordinate worldbuilding to plotting

The app should not imply that:

- plot comes first
- lore is only supporting metadata

### 2. Character import and authoring matter more than generic AI drafting breadth

The recent dogfooding supports this:

- users may arrive with rich world/character documents before they trust the app for full drafting
- import and review flows are part of onboarding into a world-first system

### 3. General Fiction should still preserve the world-first philosophy

Even outside LitRPG:

- character
- world
- canon

should remain central rather than feeling like optional secondary panels.

## Messaging Direction

Possible internal/product language:

- “A writing desk for stories where world, character, and canon carry real weight.”
- “Build the world, keep the canon straight, and write inside it.”
- “For stories where setting and character architecture matter as much as plot.”

## Near-Term Use

This note should inform:

- onboarding copy
- General Fiction mode framing
- character/world import priorities
- future positioning/messaging docs

It does not require an immediate marketing rewrite, but it should guide product decisions going forward.
